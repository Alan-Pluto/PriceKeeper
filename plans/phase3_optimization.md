# 阶段 3 — 优化与扩展（性能/质量）

目标（时长：4 周）
- 完善测试覆盖、性能基准与发布准备（以本地可重复验证为准）。

周计划
- 周 1（测试覆盖提升）：补全单元测试、UI 测试；针对关键路径（UseCase/Repository/Parser）提高覆盖率。
- 周 2（性能基准）：添加查询基准测试与导出大数据性能测试（1 万条）；识别慢查询并添加索引或改写 SQL。
- 周 3（本地自动化脚本）：编写并文档化本地测试/构建/复现脚本和步骤，提供自托管流水线建议（可选）。
- 周 4（发布与文档）：整理 Release note 模板、发布流程、打包与签名说明；进行一次模拟发布并回归测试。

交付物
- 本地自动化测试脚本与运行说明（`scripts/` 或 `docs/` 下的说明），覆盖构建与测试。
- 性能基准报告与优化方案（慢查询清单、索引建议）。
- 完整测试集与测试运行说明。
- 发布指南与版本管理说明。

验收标准
- 本地验证通过所有质量检查（lint + 单元测试 + 构建）。
- 导出 1 万条数据测试在目标时间内（≤ 5s，若平台限制则记录基线并提出改进措施）。
- 完整发布流程文档化。

AI 生成提示范例
- "生成本地测试与构建脚本（shell/gradle 命令）与运行说明，并提供常见故障排查建议。"

风险与缓解
- 在不同机器/环境波动：优先保证非 UI 的自动化检查，UI Test 在需要时再作为可选步骤运行。
