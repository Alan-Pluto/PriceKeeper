# 阶段 3 — 优化与发布准备

目标（时长：4 周）

- 提升稳定性、性能和可维护性，形成可重复验证的发布流程。

周计划

- 周 1（测试覆盖提升）：补全 UseCase、Repository、地图链接解析、导入导出和关键 Compose UI 测试。
- 周 2（性能基准）：添加商品列表、商品详情、商店详情和导出大数据基准测试，识别慢查询并优化索引或 SQL。
- 周 3（自动化脚本）：整理本地构建、测试、安装、导出导入回归脚本和常见故障排查。
- 周 4（发布与文档）：整理 Release note、签名打包流程、版本迁移说明，进行一次完整模拟发布。

交付物

- 自动化测试与运行说明。
- 性能基准报告和慢查询优化记录。
- 发布指南与版本管理说明。
- 回归测试清单。

验收标准

- `assembleDebug`、`testDebugUnitTest` 通过。
- 1 万条价格记录下，核心查询和导出性能达到可接受基线。
- 数据库迁移可以从旧版本平滑升级。
- 文档与实际代码结构一致。
