# 阶段 2 — 功能完备（位置、导入导出与体验）

目标（时长：6 周）

- 完善商店位置、地图路线规划、数据迁移、导入导出和核心页面体验，让应用从可用走向好用。

周计划

- 周 1（商店位置）：支持用户粘贴系统地图分享链接，解析高德短链和长链，保存经纬度、地址、区域与展开后的地图链接。
- 周 2（路线规划）：商品详情和商店详情统一复用地图路线规划逻辑，优先拉起高德路线规划页，失败时使用地图链接或 `geo:` 兜底。
- 周 3（导入导出）：实现 `.mypd`（gzip + json）导出导入，补齐覆盖、合并、跳过三种冲突策略和错误提示。
- 周 4（图表与聚合）：完善价格趋势图、商店比价、商店详情商品价格层级，保证 SQL 聚合与 UI 展示一致。
- 周 5（交互优化）：优化分类/商店选择弹窗、导航栏切换稳定性、商店评价输入窗格和商品信息层级。
- 周 6（QA 与文档）：补充测试、回归导入导出和地图链接解析，更新 README、PRD、TDD 与模块文档。

交付物

- 稳定的手动录入流程。
- 地图链接解析与路线规划复用组件。
- 完整导入导出实现与兼容说明。
- 商品详情、商店详情和 Profile 的数据聚合展示。
- 可执行的测试与回归清单。

验收标准

- 短链解析成功时保存经纬度、地址、区域和展开链接；解析失败时保存原始链接用于后续导航。
- 有经纬度的商店点击导航时直接进入地图路线规划页，不绕浏览器。
- 商品详情和商店详情使用同一套导航业务逻辑。
- 导入导出能完整恢复商品、商店和价格记录。
