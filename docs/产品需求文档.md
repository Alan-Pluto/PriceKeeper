# 产品需求文档（PRD）- 计价器 PriceKeeper v1.1

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 产品名称 | 计价器 PriceKeeper |
| 文档版本 | PRD v1.1 |
| 文档状态 | 当前实现基线 |
| 产品定位 | 本地优先、隐私友好的个人物价记录与比价工具 |

## 2. 产品概述

PriceKeeper 帮助用户手动记录日常商品价格、商店信息和购买时间，逐步形成个人物价数据库。用户可以查看同一商品在不同商店的价格差异、历史价格趋势，并通过保存的商店位置打开地图路线规划。

## 3. 核心理念

- 数据主权归用户：所有数据本地存储，支持完整导入导出。
- 录入稳定可控：以结构化手动录入为核心，避免自动识别误差污染数据。
- 位置轻量兼容：用户通过地图 App 分享位置链接，应用尽力解析；解析失败也保存原始链接用于后续导航。
- 决策导向：重点展示最低价、最近价、价格波动和商店维度对比。

## 4. 用户画像

- 家庭采购者：希望记住米面粮油、奶制品、日化用品等常购商品价格。
- 多平台/多渠道购物用户：希望判断当前价格是否真的划算。
- 对隐私敏感的 Android 用户：希望数据留在本机，可自行备份迁移。

## 5. 全局框架

底部导航采用四个固定 Tab：

| Tab | 页面 | 核心职责 |
|---|---|---|
| 记一笔 | 首页 | 手动录入入口 |
| 物价 | 商品看板 | 商品浏览、分类筛选、价格趋势、商店比价 |
| 商店 | 商店看板 | 商店管理、位置保存、用户评价、路线规划 |
| 我的 | 数据与设置 | 统计、导入导出、分类管理、关于 |

交互原则：

- 页面切换保持稳定，不出现导航栏闪烁。
- 子页面通过顶部返回或系统返回退出。
- 所有核心按钮触控区域不小于 44dp。
- 录入失败或解析失败时给出明确可恢复路径。

## 6. 首页：记一笔

页面定位：极简启动器，只保留手动录入主入口。

核心内容：

- 主卡片：`手动记一笔`，进入新增物价记录页。
- 信息卡片：说明当前版本专注手动录入、数据更稳定。

验收标准：

- 点击 `手动记一笔` 进入手动录入页。
- 首页只保留手动录入入口，不提供其他录入分支。

## 7. 手动录入

页面定位：快速新增单条价格记录。

字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| 商品名 | 是 | 支持输入新商品或选择历史商品 |
| 价格 | 是 | 必须大于 0 |
| 分类 | 否 | 选择已有分类或新建分类 |
| 商店 | 是 | 选择历史商店或新增商店 |
| 购买日期 | 否 | 默认当前时间 |
| 备注 | 否 | 如促销、规格、会员价等 |

商店位置录入：

- 用户可以粘贴系统地图分享链接。
- 点击解析并保存后，应用先判断长链或短链。
- 短链需要尝试重定向获取长链。
- 长链直接解析经纬度、地址、区域。
- 解析成功保存经纬度、地址、区域和展开后的地图链接。
- 解析失败保存原始链接为 `mapUrl`，后续导航直接打开该链接。

验收标准：

- 必填项为空时明确提示。
- 商品和商店支持 find-or-create。
- 地图链接解析失败不阻断价格记录保存。

## 8. 物价看板

页面定位：商品检索、分类浏览和比价决策中枢。

商品列表：

- 左侧分类栏，右侧商品列表。
- 顶部搜索框按商品名实时过滤。
- 商品卡片展示商品名、分类、最低价、最近购买信息。

商品详情：

| 模块 | 内容 |
|---|---|
| 价格摘要 | 最低价、最高价、最近价 |
| 趋势图 | 按购买日期展示历史价格 |
| 商店比价 | 按价格排序展示不同商店的当前记录 |
| 导航入口 | 对有商店位置的记录打开地图路线规划页 |

验收标准：

- 商品最低价与价格记录同步更新。
- 商店比价列表价格排序正确。
- 商品详情中的导航使用统一地图路线规划逻辑。

## 9. 商店看板

页面定位：地点管理和商店维度商品归档。

商店列表：

- 顶部搜索框按商店名过滤。
- 商店按区域分组展示。
- 商店卡片展示名称、区域、地址、追踪商品数、用户评分。

商店详情：

| 模块 | 内容 |
|---|---|
| 商店头部 | 名称、地址、区域、评分、导航按钮 |
| 用户评价 | 独立编辑入口，避免误触 |
| 商品列表 | 展示该店商品名、最近购买时间、价格区间和波动 |
| 导航 | 优先用经纬度打开高德路线规划页，失败时打开地图链接 |

验收标准：

- 用户评价输入窗格清晰、稳定，不遮挡关键操作。
- 商品信息层级清楚，价格波动突出。
- 商店详情与商品详情复用同一套导航逻辑。

## 10. 我的

页面定位：数据资产仪表盘和系统设置。

核心内容：

- 商品数、商店数。
- 导出所有数据。
- 导入数据。
- 分类管理。
- 关于与隐私说明。

导入导出：

- 导出格式：`.mypd`，GZIP + JSON。
- 导入策略：覆盖、合并、跳过。
- 当前导出内容：商品、商店、价格记录。
- 旧文件中的历史废弃字段应被忽略，不影响可导入数据。

## 11. 数据模型

核心实体关系：

```text
Goods 1 ── N PriceRecord N ── 1 Store
```

核心表：

```sql
CREATE TABLE goods (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    category TEXT NOT NULL,
    spec_unit TEXT,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);

CREATE TABLE store (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    region TEXT NOT NULL,
    address TEXT,
    latitude REAL,
    longitude REAL,
    map_url TEXT,
    my_note TEXT,
    rating INTEGER NOT NULL,
    created_at INTEGER NOT NULL
);

CREATE TABLE price_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    goods_id INTEGER NOT NULL,
    store_id INTEGER NOT NULL,
    price REAL NOT NULL,
    record_date INTEGER NOT NULL,
    is_promotion INTEGER NOT NULL,
    note TEXT,
    created_at INTEGER NOT NULL,
    FOREIGN KEY (goods_id) REFERENCES goods(id) ON DELETE CASCADE,
    FOREIGN KEY (store_id) REFERENCES store(id) ON DELETE CASCADE
);
```

## 12. 技术方案

| 层级 | 技术 | 说明 |
|---|---|---|
| 开发语言 | Kotlin | Android 主语言 |
| UI | Jetpack Compose + Material 3 | 声明式 UI |
| 架构 | Clean Architecture + MVVM | 分层清晰，便于测试 |
| 数据库 | Room | 本地唯一持久化数据源 |
| 注入 | Hilt | 构造函数注入 |
| 异步 | Coroutines + Flow | 响应式数据流 |
| 导入导出 | kotlinx.serialization + GZIP | `.mypd` 文件 |
| 地图 | Intent + 高德 URI + geo 兜底 | 不集成地图 SDK |

## 13. 非功能性需求

隐私与权限：

- 所有业务数据仅存储在设备本地。
- 不申请相机权限。
- 不采集图片。
- 不上传商品、商店、价格或位置数据到云端。
- 地图导航通过系统 Intent 交由用户设备上的地图 App 处理。

性能：

| 指标 | 目标 |
|---|---|
| 冷启动 | < 2s |
| 页面切换 | < 300ms |
| 核心查询 | < 200ms |
| 1 万条数据导出 | < 5s |

可靠性：

- 手动录入写入商品、商店、价格记录时需要保证一致性。
- 导入损坏文件时提示格式错误，不破坏现有数据。
- 数据库 schema 变更必须提供 Room Migration。

## 14. 版本规划

v1.1 当前版本：

- 手动录入。
- 商品价格趋势与商店比价。
- 商店位置链接解析与路线规划。
- 商店详情评价与商品信息层级优化。
- 本地导入导出。
- 本地导入导出。
- 不引入图片采集或自动识别录入流程。

后续方向：

- 商品条码辅助录入。
- 价格提醒。
- 桌面快捷入口。
- 更完整的数据分析。
