产品需求文档（PRD）-「计价器」v1.0

文档信息

· 产品名称：计价器
· 版本号：v1.0
· 文档状态：最终确认版
· 产品定位：本地优先、保护隐私的个人物价追踪与智能比价工具

一、产品概述

1.1 产品背景

日常生活中，用户在多家商店/平台购买相同商品时，难以记住历史价格，导致无法判断当前是否真的“买得便宜”。“计价器”通过拍照识别小票和手动录入两种方式，帮助用户建立专属的物价数据库，实现跨店比价、价格趋势追踪，让每一次消费决策都更有依据。

1.2 核心理念

· 数据主权归用户：所有数据本地存储，支持完整导入导出，不依赖任何云端服务
· 隐私第一：OCR识别全程在设备端完成，小票图片永不离开手机
· 极简高效：首页双入口，全屏沉浸操作，零学习成本

1.3 用户画像

· 精打细算的家庭采购者，希望掌握日常消费品的真实价格波动
· 多平台/多渠道购物的比价爱好者
· 对数据隐私敏感的Android用户

二、全局框架

2.1 底部导航栏

采用 Material Design 3 规范，四个Tab固定显示：

图标 标签 对应页面 核心职责
🧾 记一笔 首页 纯净入口，拍小票/手动记
📊 物价 物价看板 商品浏览，比价，距离导航
🏪 商店 商店看板 地点管理，商店归档
👤 我的 我的 数据仪表盘 + 设置

2.2 通用交互规范

· 页面切换：点击底部Tab切换，保持页面状态（返回时不刷新）
· 子页面：点击首页大卡片后进入全屏子页面，底部导航栏自动隐藏，返回时重新出现
· 转场动画：首页卡片点击使用共享元素放大转场动画

三、页面详细需求

3.1 首页：「记一笔」

页面定位：极简启动器，仅承载录入入口功能。

3.1.1 布局

· 上下或左右分栏，两大卡片铺满屏幕
· 卡片一：【📷 拍小票】—— 副标题“OCR自动识别录入”
· 卡片二：【✏️ 手动记】—— 副标题“快速添加单品价格”

3.1.2 交互流程

流程A：拍小票（全屏沉浸式三步流）

步骤 页面内容 操作
Step 1 全屏图像获取页 顶部“相册”/“拍照”Tab切换，底部大圆快门按钮
Step 2 全屏识别编辑页 展示OCR识别结果列表，每行可编辑商品名、价格、分类下拉选择；底部悬浮栏显示“识别到 X 项商品”+【确认存档】按钮
Step 3 半屏底部表单 仅询问“这次在哪家商店买的？”，输入框+历史商店快捷标签；点击“完成”回到首页

流程B：手动记（全屏极速填表）

元素 说明
顶部 标题“新增物价记录”，左上角“取消”按钮
中央表单 商品名（大输入框）、价格（自动弹出数字键盘）、分类（可搜索选择器）、商店名（历史记录联想）
底部 【保存并继续】按钮 + 【仅保存】按钮

3.1.3 边界与异常处理

· OCR识别失败：提示“未识别到有效商品信息，请尝试手动录入”，自动切至手动表单
· 必填项为空时点击保存：高亮提示“请填写商品名称”
· 重复商品提醒：若录入的商品名已存在，弹窗提示“已存在同名商品，是否继续记录？”，防止污染数据库

3.2 物价看板：「物价」

页面定位：商品检索、分类浏览、比价决策中枢。

3.2.1 布局（外卖风格——左树右列表）

· 左侧分类栏（宽度约25%）：垂直滚动列表，展示一级分类（生鲜、饮料、粮油调味、日化、自定义分类等），选中项高亮
· 右侧内容区（宽度约75%）：
  · 顶部固定搜索框，占位提示“搜商品名（如：牛奶）”，实时过滤当前分类下商品
  · 下方商品列表（垂直滚动），每个卡片展示：商品名、当前记录的最低价及对应商店、最近购买日期

3.2.2 商品详情看板（点击商品卡片进入）

模块 内容
价格趋势 历史价格曲线折线图
比价列表 按当前价格从低到高排序，展示：商店名、价格、距我X公里、标签（最便宜/最贵）
导航 右侧【导航】按钮，调用系统Intent跳转高德/百度地图

3.2.3 边界与异常处理

· 定位未开启：展示“未知距离，请开启定位”，隐藏导航按钮
· 无历史记录：展示空状态插画 + 提示“暂无价格记录，去记一笔吧”，并提供【去记一笔】跳转按钮
· 搜索结果为空：展示“未找到匹配商品”提示

3.3 商店看板：「商店」

页面定位：地点管理、商店维度商品归档。

3.3.1 布局

· 顶部：搜索框（按商店名检索）+ 右侧【添加商店】按钮
· 列表：按“省份/城市”折叠分组（如“📍 北京朝阳区”），点击展开该区下的商店卡片
· 每个商店卡片：商店名、所在地区、收录商品总数角标（如“已追踪38件”）、用户自定义星级评分

3.3.2 商店详情页（点击商店卡片进入）

模块 内容
头部 商店地址、我的备注标签、【一键导航】按钮
商品列表 该商店下所有追踪商品：商品名、当前在该店的价格、最近购买日期
底部操作栏 【编辑商店信息】、【导出该店全部记录为CSV】

3.3.3 边界与异常处理

· 某地区下无商店：折叠状态下不展示该地区分组
· 商店无关联商品：详情页展示空状态“该商店暂无商品记录”

3.4 我的：「我的」

页面定位：数据资产仪表盘 + 系统设置。

3.4.1 布局

· 顶部仪表盘区（视觉C位）：
  · 累计追踪商品数（如“128件”）
  · 覆盖商店数（如“6家”）
  · 历史总消费（如“¥3,286.5”）
  · 简约卡片风格展示
· 中部数据主权区：
  · 【📤 导出所有数据】按钮
  · 【📥 导入数据】按钮
  · 两按钮并排，强调数据的可携带性
· 底部常规设置区：
  · 深色模式开关
  · 默认分类管理（增删改）
  · 货币单位设置
  · 关于我们

3.4.2 导入导出交互

· 导出格式：.mypd（JSON + GZIP压缩），支持设置加密密码（可选）
· 导入选项：选择文件后，弹窗选择“合并”或“覆盖”模式
· 清空数据：二次确认后清空所有本地记录

4.2 建表SQL语句

```sql
-- 商品表
CREATE TABLE goods (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    category TEXT NOT NULL DEFAULT '未分类',
    spec_unit TEXT,
    current_low_price REAL,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);

-- 商店表
CREATE TABLE store (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,
    region TEXT NOT NULL,          -- 省/市/区
    address TEXT,
    latitude REAL,
    longitude REAL,
    my_note TEXT,                  -- 用户自定义备注
    rating INTEGER DEFAULT 0,      -- 用户自定义评分
    created_at INTEGER NOT NULL
);

-- 价格记录表
CREATE TABLE price_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    goods_id INTEGER NOT NULL,
    store_id INTEGER NOT NULL,
    price REAL NOT NULL,
    record_date INTEGER NOT NULL,
    receipt_img_path TEXT,
    note TEXT,                     -- 特价/原价等备注
    created_at INTEGER NOT NULL,
    FOREIGN KEY (goods_id) REFERENCES goods(id) ON DELETE CASCADE,
    FOREIGN KEY (store_id) REFERENCES store(id) ON DELETE CASCADE
);

-- 小票存档表
CREATE TABLE receipts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    store_id INTEGER,
    total_price REAL,
    buy_date INTEGER NOT NULL,
    img_path TEXT NOT NULL,
    created_at INTEGER NOT NULL,
    FOREIGN KEY (store_id) REFERENCES store(id) ON DELETE SET NULL
);

-- 创建索引优化查询性能
CREATE INDEX idx_price_records_goods ON price_records(goods_id);
CREATE INDEX idx_price_records_store ON price_records(store_id);
CREATE INDEX idx_price_records_date ON price_records(record_date);
CREATE INDEX idx_goods_category ON goods(category);
CREATE INDEX idx_store_region ON store(region);
```

五、技术方案

5.1 技术栈概览

层级 技术选型 说明
开发语言 Kotlin Android官方首选语言
UI框架 Jetpack Compose 声明式UI，高效构建
本地数据库 Room SQLite官方ORM封装
OCR引擎 华为ML Kit（端侧） 完全离线，免费，隐私安全
图表库 MPAndroidChart 价格走势曲线
图片加载 Coil 轻量级，与Compose完美配合
权限管理 Accompanist Permissions 简化运行时权限申请
导航 Navigation Compose 页面路由管理
数据导入导出 Gson + GZIP JSON序列化 + 压缩

5.2 关键实现要点

要点 方案
端侧OCR集成 集成HMS ML Kit端侧文本识别SDK，完全不依赖网络
地理位置获取 进入物价/商店页面时一次性请求定位，缓存距离结果；使用FusedLocationProviderClient
地图导航联动 拼接标准URI：geo:0,0?q=商店名+地址，调起系统Intent
数据导入导出 导出为JSON+GZIP格式，文件后缀.mypd；导入时支持“合并/覆盖”模式
深色模式 跟随系统，使用MaterialTheme动态适配
图片压缩 小票拍照后压缩至1080p宽度，减少存储占用
统计数据更新 在onResume()时重新查询数据库计算，无需实时推送

5.3 性能目标

指标 目标值
App启动时间 < 2秒（冷启动）
OCR识别速度 < 3秒（端侧，主流中端机型）
页面切换响应 < 300ms
数据库查询 < 200ms（带索引优化）
安装包体积 < 40MB（含OCR模型包）

5.4 兼容性要求

· 最低支持：Android 8.0 (API 26)
· 目标SDK：Android 14 (API 34)
· 64位设备优先，兼容32位

六、非功能性需求

6.1 隐私与安全

· 所有数据仅存储在设备本地，不上传任何云端
· 相机权限仅在拍照时申请，图片仅用于OCR识别，不持久化存储原始图（仅存压缩缩略图用于展示）
· 位置权限仅在物价/商店页面请求，可选择拒绝并手动输入地址
· 隐私政策中明确说明数据存储方式与权限用途

6.2 可用性

· 核心路径（录入手动记录）从打开App到完成存档不超过3步操作
· OCR识别结果页支持快速修正，无需重新录入全部信息
· 所有可点击区域最小尺寸 ≥ 44dp（符合Material Design触控规范）
· 空状态全部配有引导插画与文案

6.3 可靠性

· 数据库事务保证多表写入的原子性（OCR确认存档时同时写入商品、价格记录、商店三张表）
· 导入数据时做完整性校验，损坏文件提示“文件格式错误，导入失败”
· 支持Android备份/恢复机制（用户可通过系统备份保留数据）

八、验收标准

编号 验收项 标准
1 录入流程 拍小票三步完成存档；手动记两步完成存档
2 OCR识别 常见商店小票识别准确率 ≥ 85%（字段完整度）
3 比价展示 商品详情页按价格排序正确，最便宜/最贵标签准确

5 导入导出 导出文件可完整导入，数据无丢失，合并/覆盖模式正常

7 深色模式 跟随系统切换，所有页面适配无视觉死角
