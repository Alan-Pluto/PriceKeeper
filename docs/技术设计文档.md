## 2. 总体架构

```text
Presentation Layer
 ├── Compose UI
 ├── ViewModel
 ├── UiState / UiEvent / UiEffect

Domain Layer
 ├── UseCase
 ├── Model
 └── Repository Interface

Data Layer
 ├── Repository Impl
 ├── Room Database
 ├── Import / Export
 └── Map Link Parser
```

数据流：

```text
UI Event
 → ViewModel
 → UseCase
 → Repository
 → DAO / File
 → Flow / Result
 → UiState
 → Compose
```

核心约束：

- Room 是唯一持久化数据源。
- Repository 是唯一数据入口。
- ViewModel 不直接调用 DAO。
- Compose 页面只读 UiState，通过 Event 触发业务。
- 数据库 schema 变化必须提供 Migration。

    @ColumnInfo(name = "is_promotion") val isPromotion: Boolean,

## 5. 核心 SQL

商品价格摘要：

```sql
SELECT
    MIN(price) AS minPrice,
    MAX(price) AS maxPrice,
    (SELECT price
     FROM price_records
     WHERE goods_id = :goodsId
     ORDER BY record_date DESC
     LIMIT 1) AS latestPrice
FROM price_records
WHERE goods_id = :goodsId;
```

价格趋势：

```sql
SELECT record_date, price
FROM price_records
WHERE goods_id = :goodsId
ORDER BY record_date ASC;
```

商店比价：

```sql
SELECT s.name, s.address, s.latitude, s.longitude, s.map_url, pr.price, pr.record_date
FROM price_records pr
JOIN store s ON pr.store_id = s.id
WHERE pr.goods_id = :goodsId
ORDER BY pr.price ASC;
```

商店商品聚合：

```sql
SELECT
    g.id,
    g.name,
    MIN(pr.price) AS minPrice,
    MAX(pr.price) AS maxPrice,
    MAX(pr.record_date) AS lastBuyDate
FROM price_records pr
JOIN goods g ON pr.goods_id = g.id
WHERE pr.store_id = :storeId
GROUP BY g.id;
```

仪表盘统计：

```sql
SELECT COUNT(*) FROM goods;
SELECT COUNT(*) FROM store;
SELECT COUNT(*) FROM price_records;
SELECT SUM(price) FROM price_records;
```

## 9. 地图链接解析与导航

### 9.1 地图链接解析

用户输入地图分享链接后：

```text
输入链接
 → 正则判断短链或长链
 → 短链执行 HTTP 重定向展开
 → 长链提取经纬度、地址、区域
 → 成功：保存经纬度、地址、区域、展开链接
 → 失败：保存原始链接为 mapUrl
```

解析器必须满足：

- 不要求普通用户输入经纬度。
- 短链展开失败不阻断保存。
- 原始链接和展开链接都要保留可追溯性。
- 网络失败、非高德链接、异常格式均走降级保存。

### 9.2 路线规划

商品详情和商店详情统一调用 `openMapRoutePlan`：

```kotlin
openMapRoutePlan(
    context = context,
    destination = MapDestination(
        name = store.name,
        address = store.address,
        latitude = store.latitude,
        longitude = store.longitude,
        mapUrl = store.mapUrl
    )
)
```

导航策略：

- 有经纬度：优先使用 `amapuri://route/plan/` 打开高德路线规划页。
- 高德不可用：使用 `geo:` URI 兜底。
- 无经纬度但有 `mapUrl`：打开保存的地图链接。
- 无位置信息：使用商店名或地址构造通用 `geo:` 查询。

## 13. 导入导出

格式：

```text
.mypd = gzip + json
```

JSON：

```json
{
  "version": "1.0",
  "exportedAt": 1710000000000,
  "goods": [],
  "stores": [],
  "priceRecords": []
}
```

兼容策略：

- JSON 解析启用忽略未知字段。
- 旧备份中的废弃字段不参与新模型写入。
- 导入冲突支持覆盖、合并、跳过。
- 覆盖导入必须在事务内清空并写入，失败回滚。

## 14. 数据库迁移

当前数据库版本：v3。

迁移说明：

- v1 → v2：`store` 表新增 `map_url`。
- v2 → v3：重建 `price_records` 表，移除历史废弃字段，并删除旧存档表。

迁移要求：

- 保留已有商品、商店和价格记录。
- 迁移后重建索引。
- 新版本代码不再依赖历史废弃表或字段。

## 16. 测试策略

单元测试：

- UseCase 参数校验与 Repository 调用。

## 17. 最终工程约束

- 不新增图片采集或自动识别录入模块。
- 不引入 ML Kit、CameraX 或相机权限。
- Room 为唯一数据源。
- Repository 为唯一数据入口。
- ViewModel 禁止直连 DAO。
- Compose 必须单向数据流。
- 所有模块应可独立测试。
